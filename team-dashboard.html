📖 Grasp Team Dashboard — Guide
🚀 Getting Started
How to openDouble-click team-dashboard.html to open standalone in any browser — no server needed. Or run npx grasp-mcp-server in your terminal: it starts a local server on port 7331 and opens Grasp automatically. Both HTML files work the same way.
Add a repoPaste owner/repo or a full GitHub URL, press Enter or click + Add Repo
Load examplesClick "Load examples" to pre-populate with popular open-source repos
GitHub tokenOptional — paste a PAT for 5,000 req/hr and private repos. Shared with Grasp: set it in either app and both pick it up
Saved reposRepos, workspaces, and token are saved in localStorage and persist across sessions
Open in GraspClick any repo name to deep-dive into its full dependency graph in index.html
↺ Re-analyzeRe-runs GitHub analysis on all repos in the active workspace — useful after code has changed
🏢 Team Fields
Owner columnAssign a team or person responsible for each repo — click the cell to type
Status columnClick to cycle: stable → watch → in-refactor → deprecated → blocked → (clear)
Notes columnFree-text annotation per repo — click to type, saved automatically
Trend columnShows score delta vs previous analysis (+/−) and a sparkline of last 10 scores
📁 Workspaces
Workspace switcherClick the workspace name in the top-right to create, switch, or delete workspaces
Multiple workspacesEach workspace has its own repo list, owner/status/notes, and score history
⎘ ShareCopies a URL encoding the active workspace's repo list + team fields to your clipboard. Recipients can paste it in their browser to import the workspace
📊 What the Dashboard Shows
Health Score0–100 score per repo: A (90+) · B (75+) · C (60+) · D (45+) · F (<45)
Grade badgeColor-coded A–F grade for at-a-glance quality. Green = healthy, red = needs attention
Issues countTotal architecture issues: circular deps, dead code, high coupling, security findings
Circular depsNumber of circular dependency chains detected across the repo
SecurityHardcoded secrets, SQL injection risks, dangerous patterns found
LayersArchitectural layers detected (UI, Services, Data, Utils, Test, Config…)
Commits 7d / 30dCommit activity for each repo over the last 7 and 30 days — fetched from GitHub in the background after the repo loads. Falls back to cached counts if the API is unavailable
Commit sparklineA mini blue/amber SVG sparkline next to the commit counts shows commit velocity trend across your visits (up to 14 snapshots, one per session)
↺ N newAmber badge showing commits since your last Grasp analysis — click to open that repo in Grasp for a full re-analysis and diff
CI badge✅ CI / ❌ CI / ⏳ CI — pulled from GitHub Actions latest run. Green = passing, red = failing, amber = in progress or unknown
⬡ Grasp badgeShown on repos already analyzed in Grasp — deeper data (functions, circular deps, security) is available from the saved snapshot
Score bars chartVisual ranking of all repos by health score — longest bar = healthiest
Issue breakdownAggregate issue counts across all repos, grouped by issue category

🔌 MCP Server & CLI (48 tools)
MCP Servergrasp-mcp-server exposes 48 analysis tools to Claude Code and other LLM agents — run via npx grasp-mcp-server
Core toolsgrasp_analyze · grasp_hotspots · grasp_cycles · grasp_security · grasp_refactor · grasp_pr_comment · grasp_migration_plan · grasp_api_surface
grasp_env_varsScans files for env var reads, cross-references .env.example, flags undocumented and test-only vars
grasp_eventsMaps event emitters/subscribers, detects orphaned and ghost events
grasp_staleFinds active but abandoned files — low churn, high fan-in, no test. Returns staleness score 0–100
grasp_change_riskRisk score 0–100 for a set of changed files — blast radius, complexity, churn, and layer violations
grasp_feature_flagsFinds all feature flag reads — LaunchDarkly, GrowthBook, OpenFeature, env-var, and custom patterns
grasp_perfDetects N+1 queries, sync I/O calls, and JSON serialization inside loops
grasp_onboardOrdered reading path for new engineers entering an area of the codebase
grasp_licenseScans node_modules for dependency licenses — reports permissive, copyleft, and unknown
grasp_typesType annotation coverage per file — prioritizes high fan-in files lacking types
grasp_diagramGenerates Mermaid flowchart or C4 diagrams — context, container, or component level
grasp_pr_reviewPosts inline review comments on GitHub PRs at high-severity lines
grasp_suggestEnhanced — now includes effort-to-impact ratio, sorted best ROI first
🔒 Privacy
100% browser-basedAll analysis runs in your browser — no backend, no data sent anywhere
GitHub APICalls go directly to api.github.com. Without token: 60 req/hr. With PAT: 5,000 req/hr
Token storageYour GitHub token is stored in localStorage only — it never leaves your browser. Shared with Grasp so you only need to set it once
