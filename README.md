## What is Grasp?

**Grasp** turns any GitHub or GitLab repository тАФ cloud or self-hosted тАФ or local codebase into an interactive architecture map in seconds. **121 MCP tools** (plus 8 Resources and 2 guided Prompts) expose the full analysis engine to Claude Code, Cursor, and any MCP-compatible agent.

```
Paste URL / Open Folder  тЖТ  AST Analysis Engine  тЖТ  Architecture Map + 121 MCP Tools
```

| | |
|---|---|
| **No installation** | Runs 100% in your browser тАФ two HTML files, no build step |
| **No data collection** | Your code never leaves your machine |
| **No accounts** | Paste a URL and go |
| **Works offline** | Analyse local folders without internet |
| **35 languages** | JS/TS, Python, Go, Java, Rust, C/C++, C#, Ruby, Swift, Kotlin, Scala, Dart, Elixir, Erlang, Haskell, OCaml, F#, Clojure, Julia, Lua, R, Perl, Shell, PowerShell, Groovy, Zig, V, Nim, Crystal, VBA, Ada/SPARK, Vue, Svelte, PHP |
| **121 MCP tools** | Dependency graphs, security, **OSV.dev SCA vulnerability scanning**, DORA, brain store, Kuzu graph schema v3, communities, ORM tracker, git change impact, architecture drift detection, test coverage gap map, org dashboard, PR impact action, MCP Resources/Prompts, `grasp setup` editor auto-config |
| **12 AI providers** | Anthropic Claude, OpenAI GPT-4o, Google Gemini, Mistral, Groq, DeepSeek, Ollama, OpenRouter, Together, Azure OpenAI, LM Studio, plus any local OpenAI-compatible endpoint |
| **10 graph views** | Force graph, 3D, arch, treemap, matrix, tree (dendrogram), flow (sankey), bundle, cluster (disjoint), heatmap |
| **Grasp Brain** | SQLite + Kuzu persistent store тАФ index once, query instantly. FTS5 + 384D vector embeddings + Cypher graph queries |
| **Supply chain signed** | SLSA Level 2 npm provenance + Cosign keyless Docker signing on every release |

```bash
git clone https://github.com/ashfordeOU/grasp.git
open index.html           # Main app
open team-dashboard.html  # Team Dashboard
```

No build step. No `npm install`. **Two HTML files.**

### Option 2 тАФ CLI

```bash
npm install -g grasp-mcp-server

grasp ./my-project        # Analyse a local folder
grasp facebook/react      # Analyse a GitHub repo
grasp .                   # Analyse current directory
grasp . --watch           # Live mode тАФ browser reloads on every file save
grasp . --timeline        # Time-travel тАФ last 30 commits as a scrubber
grasp . --report          # Terminal-only report + JSON output
grasp . --format=sarif    # Export SARIF for GitHub Code Scanning
grasp . --pr-comment      # Print GitHub PR comment markdown to stdout
grasp . --check           # Enforce grasp.yml architecture rules (CI gate)
```

## Code Intelligence

### ЁЯУК Health Score
Instant **AтАУF grade** based on dead code, circular dependencies, coupling metrics, and security issues. Displayed as a score (0тАУ100) with a visual bar.

### ЁЯФР Security Scanner
Automatic detection of hardcoded secrets & API keys, SQL injection risks, dangerous `eval()` usage, and debug statements left in production.

### ЁЯЫбя╕П Dependency Vulnerability Scanner *(v3.17.0)*
Scans declared dependencies against the [OSV.dev](https://osv.dev) free public CVE database тАФ every analysis. Supports `package.json` (with `package-lock.json` resolution), `requirements.txt`, `pyproject.toml`, `go.mod`, `Cargo.toml` (with `Cargo.lock` resolution), and `pom.xml`. Severity-classified results with CVSS scores and fix-version suggestions. New **VULN** tab in the right panel; new `grasp_vulnerabilities` MCP tool; new `grasp vulns <path>` CLI that exits 1 on critical/high findings (CI-friendly). Health score deducts тАУ5 per critical and тАУ3 per high. **100% client-side** тАФ OSV requests go directly from your browser to OSV.dev, never through a Grasp server. 24-hour localStorage cache; degrades silently on network failure.

### ЁЯзй Pattern Detection
Identifies Singleton, Factory, Observer/Event patterns, React hooks, and anti-patterns (God Objects, high coupling) тАФ automatically.

### ЁЯТе Blast Radius Analysis
*"If I change this file, what breaks?"* тАФ select any file and see every downstream file that would be affected, highlighted on the graph.

### ЁЯФе Activity Heatmap
Colors files by commit frequency. Works for GitHub repos (via API) and **local repos** (via `git log` тАФ no internet required).

### ЁЯФО Duplicate & Similarity Detection
The **Dup** color mode highlights files with exact or near-duplicate code. The `grasp_similarity` MCP tool returns ranked duplicate clusters for targeted refactoring.

### ЁЯСе Code Ownership
Top contributors per file from git history, with line-percentage breakdowns. One-click jump to GitHub Blame.

### ЁЯУЛ PR Impact Analysis
Paste a PR URL to see which files it touches and calculate the blast radius of proposed changes before merging.

### ЁЯТ░ Technical Debt Quantification
Converts every architectural issue into developer-hours using configurable estimates тАФ circular dep = 4h, god file = 16h, critical security = 8h тАФ with a coupling multiplier. Shown in the health panel and Team Dashboard.

### ЁЯФЧ Shareable Embeds
Click `тЛп тЖТ ЁЯФЧ Embed` for a ready-to-paste `<iframe>`, README badge, React snippet, and direct link тАФ share live health reports in docs, wikis, or dashboards.

### ЁЯОп Connection Confidence Scoring *(v3.16.0)*
Every cross-file connection is scored 0тАУ1: explicit static imports = 1.0, same-folder = 0.8, cross-folder inferred = 0.6, low-frequency = 0.4. The force graph overlays confidence as edge opacity тАФ use the slider in тЪЩ settings to filter out low-confidence edges.

### ЁЯФН Graph Query Modal *(v3.16.0)*
Click the ЁЯФН toolbar button to search files, functions, and edges in-browser without leaving the graph. Matches update live тАФ click any file result to jump to it on the graph.

### ╞Т() Function-Level Canvas *(v3.16.0)*
Toggle the `╞Т()` button to switch the force graph from file-level to function-level nodes тАФ see individual function call relationships, capped at 300 nodes for performance.

### ЁЯЧДя╕П DB Coupling Tab *(v3.16.0)*
The right panel **ЁЯЧДя╕П DB** tab scans file content for ORM patterns (Django, TypeORM, raw SQL), mapping which files reference which tables. Instantly spot god-tables and high-coupling files.

### ЁЯОп Good First Issues Tab *(v3.16.0)*
The **ЁЯОп GFI** tab surfaces isolated, low-complexity, untested files тАФ ideal contribution targets for new engineers or AI coding agents.

### ЁЯФР PII Detection & Security Subcategories *(v3.16.0)*
The Security tab now has subcategory pills тАФ **ALL / SECRETS / INJECTION / PII / EVAL** тАФ to filter findings. The PII pill scans file content for email, phone, SSN, credit card, and API key patterns in source files.

### ЁЯУ╕ Architecture Drift Detection *(v3.17.0)*
Snapshot your codebase architecture and detect drift over time тАФ automatically.

```bash
grasp snapshot ./my-project --name before-refactor
# ... make changes ...
grasp drift ./my-project          # exits 1 if drift is CRITICAL (CI-friendly)
```

| MCP Tool | Description |
|----------|-------------|
| `grasp_snapshot` | Save current health score, coupling metrics, circular deps, and top-10 hotspots as a named snapshot |

## Grasp Brain тАФ Persistent Architecture Intelligence *(v3.16.0)*

Grasp Brain combines two persistent stores that work together:

- **SQLite Brain** (`~/.grasp/brain.db`) тАФ file metadata, coupling, security, and issue index. Includes a FTS5 full-text index over functions and an in-process 384D vector embedding store (Xenova/all-MiniLM-L6-v2 тАФ no cloud dependency). Index once, query instantly.
- **Kuzu Graph DB** (`~/.grasp/graph/`) тАФ native graph database with Cypher query support. Stores the full function call graph, file imports, and type relationships as a traversable property graph.

Index once, then query instantly тАФ no re-analysis needed. Every function is tagged with the execution processes it participates in (BFS from entry points), so search results include a `processes[]` field grouping matches by flow.

### How it works

```
grasp index ./my-project    тЖТ  analysis stored in ~/.grasp/brain.db
grasp context src/api.ts    тЖТ  instant file context from the stored index
grasp diff ./my-project     тЖТ  compare current state vs stored baseline
grasp daemon ./my-project   тЖТ  watch for changes, re-index automatically
```

### CLI Subcommands

```bash
grasp index <path>           # Analyse and persist a repo to the brain
grasp context <src> <file>   # Get rich context for any file
grasp setup [path]           # Install hooks in Claude Code / Cursor / Windsurf
grasp diff <path>            # Compare current analysis vs brain baseline
grasp daemon <path>          # Watch directory and auto-reindex on changes
grasp drift [path]           # Snapshot + diff vs last snapshot; exits 1 on CRITICAL
grasp org <github-org>       # Org-level dashboard (--format json|html|md --token ghp_xxx)
```

### CLI-based CI Gate


```yaml
- name: Grasp health gate

    if [ "$PASSED" != "true" ]; then
      cat grasp-report.json | jq '.ci.failures'

    fi
```

## Version & Auto-Update

Both `index.html` and `team-dashboard.html` display the current version (`v3.17.1`) in the footer. On load, they silently check the npm registry for a newer release. If found, a dismissible toast appears:

- **Update Now** тАФ fetches the new HTML from GitHub, downloads it, and applies it immediately
- **Later** тАФ snoozes for 24 hours

No server, no background process.

## Privacy & Security

**Your code stays on your machine.**

**Browser app:**
- Runs 100% in the browser тАФ no server, no proxy
- GitHub/GitLab API calls go directly from your browser to the provider
- Your token lives in `localStorage` only тАФ never sent anywhere except the chosen Git provider
- No analytics, no tracking, no accounts
- The entire app is [one open-source HTML file](index.html) тАФ audit it yourself

**MCP server:**
- Runs locally as a subprocess тАФ no outbound connections except the GitHub/GitLab API
- No telemetry, no data collection
- Local directory analysis is read and discarded in memory; Brain store stays on your machine at `~/.grasp/brain.db`

**Supply chain:**
- Every npm release is signed with [SLSA provenance](https://slsa.dev) (Level 2) via GitHub Actions OIDC
- Every Docker image (`ghcr.io/ashfordeou/grasp`) is signed with Cosign keyless signatures, recorded in the [Sigstore Rekor](https://rekor.sigstore.dev) public ledger

Verify before installing:

```bash
# npm package
npm install -g @sigstore/verify  # one-time
sigstore verify npm grasp-mcp-server@3.17.1

# Docker image
cosign verify \
  --certificate-identity-regexp="https://github.com/ashfordeOU/grasp/.github/workflows/publish.yml" \
  --certificate-oidc-issuer="https://token.actions.githubusercontent.com" \
  ghcr.io/ashfordeou/grasp:v3.17.1
```
